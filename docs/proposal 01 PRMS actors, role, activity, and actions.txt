Project, and research management system - user actors, and tasks
1. Student
* Authentication & registration
Submit registration number for first-time access
Log in with temporary password on first login
Change temporary password (forced on first login)
Log in with registration number and password (normal login)
Reset password if forgotten
* Project/research activities (project/research students only)
View assigned supervisor details
Upload project artifacts, proposals, reports, demos, and source code
Download submitted and approved files
View supervisor comments and feedback
Track project status through stages
* General activities (all students)
View previous projects/research (read-only)
Download previous projects (read-only)
* Notifications
Receive temporary password via email/phone
Receive notifications for supervisor comments
Receive notifications for approval status changes

2. Coordinator
* Authentication
Log in using system credentials
* User management
Create student accounts manually (if needed)
View all active students (filtered by department and program)
See only eligible students (active, in project/research year)
* Project/research management
Create project groups (2+ students)
Register individual research students
Assign supervisors to groups/individuals
View all groups and students
Track progress of all projects/research
View approved groups ready for presentation
Access archived projects/research
3. Supervisor
* Authentication
Log in using system credentials
* Project/research management
View assigned work
View assigned project groups
View assigned individual research students
View student details, name, registration number, phone number, email
* Review & evaluation
View submitted materials, proposals, reports, demos, source code, add comments and feedback, approve or reject project stages, digitally approve final presentation forms
* Tracking
Track progress for each assigned project
* Notifications
Receive email notifications for new submissions
3. Sis (external system)
* Data provision
Serve as authoritative student data source
Provide student records via api/file export
Provide enrollment status updates
